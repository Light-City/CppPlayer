## 2.开发规范/提交规范


```
git commit -m "cpp: xxx --content"

git commit -m "cpp: xxx --bugfix"
```

这里给出两种提交格式，第一个是提交内容，每次提交把xxx改为相关修改即可，例如本次修改时更新了readme，那么就是：
```
git commit -m "cpp: update readme --content"
```
如果是修复bug，把`--content`改为`--bugfix`
